## Operating Rules

- public simplicity is a product rule, not a creative limit
- internal repertoire expansion is the main ambition rule
- untouched autonomous runs remain the primary success test
- each family must prove quiet, impact, and aftermath value
- a family graduates only with evidence
- no new lasting repertoire family belongs in legacy monolithic ownership
- a facade does not count as extraction
- a namespace shim does not count as ownership

## Graduation Ladder

The exact rules live in [graduation-rubric.md](C:/dev/GitHub/visulive/docs/graduation-rubric.md).

Short version:

- `lab`: experimental internal family
- `frontier`: eligible in frontier no-touch runs
- `flagship`: allowed in the default public untouched show
- `retired`: rejected, merged, or superseded

## Do Not Drift Into

- hero monopoly
- decorative chamber
- fake escalation by brightness
- permanent residue
- mixed-media collage behavior
- empty novelty without consequence
- strong metadata with weak visible change
- safe-tier treated like a degraded backup
- optimistic extraction claims that are not true in code

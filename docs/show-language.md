# VisuLive Show Language

Date: 2026-04-22
Status: Active artistic language brief

This document defines the visual, emotional, and behavioral language of the current product.

It exists so the project does not drift into:

- effect accumulation
- slider-led design
- random spectacle
- contradictory scene behavior

This is the director-level grammar for the system.

## The Core Reframe

VisuLive is not just a visualizer.

It is a **performance system**.

The visual should feel like:

- a living artifact
- a portal engine
- a pressure chamber
- a haunted stage entity
- a dreamlike machine with taste
- a neon ritual machine when the music earns it

The key is not constant reactivity.
The key is **selection, memory, and consequence**.

The show must be **consistently satisfying**, not occasionally amazing by accident.

The show should feel **directed from within**, not merely steered by exposed controls.

That means color, framing, chamber activation, laser drive, and radiance should be able to drift and intensify over time with the music even when the operator touches nothing.

The active reference translation layer is [live-show-reference-patterns.md](C:/dev/GitHub/visulive/docs/live-show-reference-patterns.md). Use it for real-world live show patterns, but treat the conclusions as VisuLive-specific engineering rules, not as permission to copy another show's surface style.

## Anthology Rule

VisuLive is one public portal into a larger anthology engine.

That means:

- the public start path stays simple
- the internal image vocabulary should keep getting broader
- new grammars should change the ontology of the show, not just restyle one fixed answer
- internal growth should increase no-touch greatness instead of increasing public complexity

## Whole-Frame Authority

The whole screen must participate.

The image should not read as one centered object with decorative support.
At different moments the main authority should shift between:

- the hero
- the chamber
- the environmental field
- the accent / tactical event layer
- the post / residue layer

The hero can stay iconic.
It cannot monopolize authorship.

## Stage-Led Motion

Motion should read as spatial phrase work, not one-axis wobble.

The show should be able to:

- bank when the cue asks for tilt
- roll when the cue earns camera authority
- arc and precess when the chamber needs a wider read
- tumble or recoil when the event wants more consequence

Slow passages still need visible travel.

The goal is not to spin everything harder.
The goal is to make the movement feel deliberate in full space.

## Whole-Frame Tests

A frame is doing its job if a viewer can answer these questions quickly:

- what layer is in charge right now
- is the image gathering, spending, or recovering
- did the cue change the frame or only the object brightness
- does the room feel wider, deeper, or heavier than before
- does the aftermath leave a visible trace

If those answers are unclear, the frame is under-authored.

## Screen-Space Consequence

VisuLive cannot rely only on 3D object behavior.

The show needs image-space consequence that survives beyond the hero mesh.

That means selective use of:

- bloom
- afterimage
- residue
- masked flare
- chromatic stress on impact windows
- directional stains
- beam ghosts
- ring ghosts
- local luma burns
- event-shaped wipe logic

Rule:

A major event should change not only object values, but the image itself.

The screen should remember what just happened.

## Cue Classes

The show should not resolve every moment as one generic response.

It should resolve events into cue classes such as:

- `gather`
- `tighten`
- `reveal`
- `fan-sweep`
- `orbit-widen`
- `laser-burst`
- `rupture`
- `collapse`
- `hold`
- `residue`
- `recovery`
- `haunt`

Cue classes should be visually distinct and reusable.

They are the bridge between conductor truth and scene consequence.

## Hero Anti-Monopoly Rule

The hero cannot monopolize every important frame.

At times the chamber, residue, beam geometry, or environmental stain must become the dominant read.

If the eye only remembers the center object, the frame is under-performing.

## Hero Ecology

The anthology target requires more than one hero grammar.

The show should be able to spend:

- dominant hero
- twin hero
- fractured hero
- membrane or lattice hero
- glyph or ghost hero
- absent hero or world-as-hero

The hero can remain iconic.
It cannot remain singular forever.

## World Mutation

Worlds should not only be selected.
They should also be able to mutate.

Target mutation verbs include:

- open
- seal
- flood
- hollow
- fracture
- veil
- stain
- burn
- blackout

These changes must be driven by music and cue logic, not timer-only cycling.

## The Audience View

Always design for three viewing distances.

### 1. Across the room

What must read:

- silhouette change
- large compositional shifts
- pressure / release
- major scene consequences
- calm versus active posture

### 2. Mid distance

What should read:

- chamber behavior
- world participation
- rhythm and event timing
- macro contrast
- cue identity

### 3. Up close

What should reward:

- membranes
- ghost lattice traces
- internal cavities
- particle and field detail
- glass/light/material nuance

Macro readability must always beat micro cleverness.

## The Screen Is The Stage

Think of the scene in layers:

### Far background world

- void
- stain
- chamber dark
- distant color field

### Environmental field

- haze
- dust
- veils
- drift
- afterimage

### Structural / chamber layer

- portal rings
- vaults
- lattice
- pressure architecture
- laser fans
- giant chroma halos
- stage-scale guide geometry

### Hero entity

- shell
- nucleus
- cavity
- crown
- twin / split variants

### Accent layer

- shards
- satellites
- pressure waves
- tactical punctuations

### Post / residue layer

- bloom
- afterimage
- stain residue
- light ghosts

At times, the background or chamber should become the main event.

## Color Language

The palette is no longer limited to restrained gold and teal.

The current target also includes:

- laser cyan
- tron blue
- volt violet
- hot magenta
- acid lime
- matrix green
- solar orange
- electric white flashes
- toxic pink
- cyber yellow

The palette should spend as:

- laser fan sweeps
- chroma halos
- portal stains
- matrix / acid transient pops
- toxic-pink and cyber-yellow ignitions

Rule:

- keep the blacks valuable
- let color spend feel earned
- when the music surges, the screen should go electric instead of staying tasteful by habit
- the hero cannot hide in a single safe purple blend
- additive layers should look like emitted light, not merely tinted surfaces
- the hero should read as a dark anchor wrapped in rim light, seam light, and fresnel energy before it reads as a bright filled volume
- ambient glow must stay capped and dark enough that event glow feels like a spend, not permanent fog
- the hero cannot behave like a permanent close-up; the chamber must keep enough negative space and scale contrast that the frame still reads as a world

Current authored palette states:

- `void-cyan`
- `tron-blue`
- `acid-lime`
- `solar-magenta`
- `ghost-white`

These states should hold long enough to read as directed choices, then hand off on section changes, releases, and strong phrase turns.
Palette changes should be authored by act and family, not by momentary signal twitch alone.

Current hidden show acts:

- `Void Chamber`
- `Laser Bloom`
- `Matrix Storm`
- `Eclipse Rupture`
- `Ghost Afterimage`

These are not public scene buttons.
They are internal act families that let the flagship transform meaningfully without becoming random scene soup.

## Motion Grammar

The system should be designed as a set of verbs.

Not generic wobble.
Not random drift.

Primary verbs:

- inhale
- exhale
- gather
- bloom
- drift
- hover
- float
- shear
- split
- peel
- orbit
- recoil
- collapse
- rupture
- stain
- haunt
- tunnel
- eclipse
- reassemble

Time-aware motion between beats now matters as much as beat hits.

The current internal windows are:

- pre-beat lift
- beat strike
- post-beat release
- inter-beat float
- bar turn
- phrase resolve

The shell and chamber should spend those windows differently:

- pre-beat lift tightens and leans
- beat strike punctuates
- post-beat release peels and recoils
- inter-beat float lets the world breathe
- bar turn and phrase resolve are allowed to reorganize space and palette

These verbs are the real design primitives.

## Memory And Recurrence

The show should remember itself.

That means:

- motifs return altered
- scars persist
- worlds revisit earlier states with consequence
- color and shape signatures recur with intent

Recurrence should feel authored, not repetitive.

## Mixed-Media Discipline

Mixed media is allowed if it makes the show stronger.

That does not mean collage chaos.

Allowed mixed-media behavior should feel like:

- masks
- glyphs
- screen-space scars
- silhouette injections
- portal reveals
- residue echoes

Mixed media must serve consequence, memory, or stage identity.
It must not become wallpaper.

## Emotional Arc Grammar

The system should be able to move through emotional states, not just audio amplitudes.

Core emotional sequence:

- hush
- atmosphere
- cadence
- tactile
- generative
- surge
- aftermath

These already align with the current `showState` model in code, but this doc defines how they should feel, not just what they are named.

### Hush / Void

- sparse
- suspended
- spectral
- expensive
- almost immobile, but not dead

### Atmosphere

- the room is present
- the field wakes
- the chamber breathes
- attention is gathering, not yet spending

### Cadence

- speech-like articulation
- focus
- elegance
- no chaos

### Tactile

- impact
- punctuation
- pressure rings
- shard flicks
- quick glints

### Generative

- the piece is now musically inhabited
- the world organizes
- rhythm and body start cooperating

### Surge

- the system commits
- larger consequences
- stronger structure
- special moments become available

### Aftermath

- linger
- ghost
- stain
- memory
- less volume, more consequence

## Signature Visual Behaviors

These are the recurring motifs the system should build around.

### 1. Pressure from within

The hero should often feel like it is containing more than it should.

This is one of the strongest emotional signatures available to the system.

This is strongest when the outer shell does not simply follow the core.
The eye should feel:

- drag
- resistance
- delayed bloom
- layered release
- slight parallax between inner and outer structure

### 2. Architecture waking behind the entity

The chamber should sometimes reveal itself as:

- a portal
- a vault
- a haloed machine
- a dimensional frame

This is how the system escapes "blob in center" syndrome.

### 3. Musical deepening, not just brightening

Under low and medium music, the piece should:

- deepen
- organize
- accumulate
- reveal

Not just become more luminous.

### 4. Rare earned moments

The biggest moments should feel:

- authored
- sparse
- memorable

Never frequent enough to become wallpaper.

### 5. Repeatable archetypes

When similar musical moments happen, the system should tend to resolve them into similar visual classes.

Examples:

- a heavy, sub-led drop should reliably feel like collapse, pressure, rupture, or portal authority
- a cleaner structural build should reliably feel like chamber alignment or cathedral-scale opening
- a release should reliably leave residue, ghost, stain, or hush consequence

Variation is still good.
But it must come from musical character, not from arbitrary sequencing.

## Anti-Goals

The system must not become:

- noise soup
- rainbow equalizer energy
- constant motion everywhere
- generic EDM graphics
- over-post-processed mush
- a debug UI with art attached

Specific anti-goals:

- never let every layer scream at once
- never erase negative space
- never make impacts read like random jitter
- never make similar drops feel awesome once and weak the next time for no good reason
- never make silence feel dead
- never let the operator need ten controls to get one good result
- never lock the scene into one-axis motion when the cue wants broader spatial travel

## Display Context Bias

The same visual system should be capable of three different postures.

### Desk

- more detail
- less macro aggression
- readable in a window

### Living room / TV

- wider composition
- stronger negative space
- clearer chamber participation

### Event / venue

- bigger consequences
- stronger silhouette transitions
- clearer large-scale motion and event spending

This should mostly be handled by tuning and composition bias, not separate products.

## What The Next Artistic Pass Should Target

The next big artistic improvement should emphasize:

- stronger background/world participation
- clearer contrast between hush, generative, surge, and aftermath
- more disciplined event spending
- more distinct motion phrases and camera reads
- more consistent musical archetypes for drops, builds, and aftermath
- clearer palette holds and handoffs by act and family
- more legible composition from far away
- fewer but stronger signature moments

That is more important than adding many more effects.

## Bottom Line

The system is strongest when it feels:

- withheld
- selective
- heavy with consequence
- occasionally transcendent
- layered enough that the eye keeps discovering depth and separation inside the same moment

That is the real show language.

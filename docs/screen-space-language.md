# Screen-Space Language

Date: 2026-04-08  
Status: Active reference language

This doc defines how VisuLive should spend on image-space consequence.

Screen-space is a first-class stage language, not garnish.

It should be selective, cue-authored, and readable from across a room.

## Core Rule

Major events should change the image, not only the object.

If the same cue can happen without altering bloom, residue, stain, wipe, or frame memory, then the post language is too weak.

## Allowed Spend

Screen-space effects should be used as earned consequence:

- bloom
- afterimage
- residue
- stain
- scar
- wipe
- ghost outline
- directional flash
- chromatic strain
- local luma burn
- subtractive carve

These effects should not all fire at once.

## Spend Discipline

The compositor should obey these rules:

- spend brightness only when the cue earns it
- suppress additive washout before raising exposure further
- prefer subtraction or carving when the cue calls for tension, not celebration
- let aftermath read as altered space, not as a blur cloud
- keep the blacks valuable

## Cue-Family Mapping

### Gather

Screen-space should:

- hold back
- tighten
- precharge
- prepare a later spend

Avoid:

- early bloom flood
- fake impact before the event

### Reveal

Screen-space should:

- open the frame
- use directional wipe or clean lift
- expose architecture without flattening it

Avoid:

- over-whitening the whole image

### Rupture

Screen-space should:

- flash with intent
- carve a clear before/during/after difference
- leave a visible trace

Good rupture residue:

- scar
- stain
- ghost edge
- energy wake

Avoid:

- full-frame milkiness
- long washout
- bloom that erases silhouette

### Release

Screen-space should:

- exhale
- soften
- trail
- recover

Avoid:

- immediate dead reset
- no visible tail

### Haunt

Screen-space should:

- remember
- linger
- leave delayed traces
- favor spectral residue over flash

Avoid:

- bright impact language
- heavy bloom that hides the memory

### Reset

Screen-space should:

- clear
- reduce
- reframe
- make the next state legible

Avoid:

- keeping all previous residue alive just for prettiness

## Image-Class Criteria

The screen-space pass is working when at least one of these reads clearly:

- `Impact Flash`: short, decisive, and attached to a real event
- `Directional Wipe`: a framed transition that reorders the image
- `Afterimage Wake`: a trailing memory that still preserves structure
- `Energy Scar`: a persistent altered region that marks what happened
- `Ghost Residue`: a quiet delayed trace that favors atmosphere over brightness
- `Subtractive Carve`: a negative-space response that makes tension readable

## Anti-Goals

Screen-space must not become:

- permanent haze
- constant bloom fog
- a safety blanket for weak staging
- an always-on smear pass
- a brightness contest
- a way to hide the fact that the whole frame did not change

## Readable Test Questions

Use these questions during review:

- Did the cue alter the image, or only increase brightness?
- Is the post selective and earned, or always on?
- Does aftermath leave a readable memory?
- Can the frame still read at distance after the effect?
- Does screen-space help the stage read better, or just look louder?

# VisuLive Doc Upgrade Addendum

Date: 2026-04-08  
Status: targeted doc surgery to make Codex materially better

## Why this exists

The current docs are good enough to preserve direction, but not yet sharp enough to force elite creative execution.

The main gap is:
- they describe taste and direction well
- they do **not yet enforce enough explicit on-screen behavior, cue grammar, compositing expectations, and anti-goals**

This addendum is the doc surgery pass that should happen before or alongside the next major Codex round.

---

# 1. `docs/show-language.md` — strengthen the visual grammar

## Keep
- performance system framing
- selection / memory / consequence language
- audience-distance framing
- layer stack model
- dark-value discipline
- emitted-structure hero direction

## Add explicitly

### A. Screen-space consequence
Add a new section:

```md
## Screen-Space Consequence

VisuLive cannot rely only on 3D object behavior.

The show needs an image-space vocabulary for:
- selective bloom
- afterimage
- event-localized flare
- masked chromatic stress
- luma stains
- environmental residue
- beam and ring ghosts

Rule:
A major event should change not only object values, but the image itself.

The screen should remember what just happened.
```

### B. Cue classes
Add a new section:

```md
## Cue Classes

The show should not react as one generic continuum.

It should resolve events into cue classes such as:
- inhale
- tighten
- reveal
- fan sweep
- orbit widen
- laser burst
- rupture
- collapse
- blackout hold
- residue stain
- recovery drift

Cue classes should be visually distinct and reusable.
```

### C. Hero anti-monopoly rule
Add:

```md
## Hero Monopoly Rule

The hero cannot monopolize every important frame.

At times the chamber, residue, beam geometry, or environmental stain must become the dominant read.

If the eye only remembers the center object, the frame is under-performing.
```

### D. Anti-goals
Add:

```md
## Explicit Anti-Goals

Do not drift into:
- tasteful orb demo energy
- sculpture lighting as default logic
- permanent bloom fog
- always-centered composition by habit
- random rainbow spectacle
- tiny reactive noise everywhere
- “same scene but louder” as the main form of escalation
```

# 3. `docs/vision-ledger.md` — make it sharper, not longer

## Keep
- high-signal ideas only
- promotion rules
- questions worth keeping open

## Add these new ledger items

### A. 2D / 3D hybrid consequence

```md
### Hybrid image-space consequence

The show likely needs a stronger hybrid between 3D scene rendering and screen-space event compositing.

Open direction:
- selective bloom zones
- afterimage residue
- masked chromatic impact
- luma-driven environmental stains
- beam/halo image ghosts

Reason:
Some desired spectacle may not be reachable through hero/chamber geometry and material tuning alone.
```

### B. Cue engine above conductor

```md
### Cue engine above conductor

The conductor should not be the final artistic authority.
A higher show layer likely needs to translate musical detections into named cue classes, holds, transitions, and aftermath states.
```

### C. Chamber supremacy moments

```md
### Chamber supremacy moments

Some of the strongest future frames may be chamber-dominant rather than hero-dominant.
The system should explore moments where rings, vaults, beam planes, stains, or void architecture temporarily become the main event.
```

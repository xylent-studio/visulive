# VisuLive North Star Pack

Date: 2026-04-08  
Status: Proposed creative and systems brief for the next Codex / subagent cycle

---

## Why this exists

VisuLive does **not** mainly need more coding power.
It already has that.

What it needs now is a stronger **creative operating system** above the coding.

The current repo has real infrastructure, real listening, real direction layers, real evidence capture, and real momentum.
What it does **not** yet have at the level it needs is:

- a brutally clear north star for what “great” actually means on screen
- a stronger map of references by **principle**, not just by name
- a show-cue grammar that clearly separates detection from consequence
- stronger acceptance criteria for composition, authority, pacing, and spectacle
- a better briefing layer for Codex, so it is not forced to invent taste while also writing code

This file is meant to solve that.

---

## The true target

VisuLive is not trying to become a nice browser visualizer.

It is trying to become:

**a room-commanding, music-driven stage machine**

That means:

- something that can sit on a TV at a party and instantly feel premium, alive, and impossible to ignore
- something that can scale emotionally toward the feeling of concert visuals, VJ systems, laser shows, boss-fight VFX, and authored music experiences
- something that feels like it has memory, pressure, escalation, release, residue, and intent
- something that can be mesmerizing in silence between hits and devastating when the music earns a spend

The wrong target is:

- a tasteful orb
- a shader demo
- reactive wallpaper
- constant pretty motion
- endlessly tweaking one center object

The right target is:

- an authored show system with image hierarchy
- consequence at multiple timescales
- strong black-value discipline
- iconic visual grammar
- stage-scale screen participation
- emotional pacing

---

## The experiential promise

A good VisuLive show should make a viewer feel all of the following at different times:

- **presence** — there is something alive in the frame
- **pressure** — the system is gathering or containing force
- **consequence** — events change the image, not just intensify it
- **memory** — drops and turns leave residue, ghosts, stains, or altered posture
- **authorship** — the show feels directed, not merely reactive
- **spectacle** — there are real moments that would make people look up and go “whoa”
- **mystery** — not everything is exposed or literal

If the output is technically reactive but does not deliver these feelings, it is missing the goal.

---

## What the show should feel like at three scales

### 1. Across the room

The whole frame must read immediately.

At distance, the viewer should clearly feel:

- whether the system is dormant, gathering, pulsing, rupturing, or haunting
- whether the frame is tightening or widening
- whether the hero is dominant or the chamber/world has taken over
- whether the music is in tension, impact, release, or aftermath

This is where most “pretty but weak” visualizers fail.
They have detail but not authority.

### 2. At couch / party distance

The viewer should notice:

- large movement logic
- cue timing
- layer participation
- phrase-scale transitions
- color spends that feel earned
- image-space aftermath and residue

### 3. Up close

The viewer should be rewarded with:

- structural seams
- lattice energy
- shell intelligence
- micro ghosts and afterimages
- material nuance
- hidden field behavior

Rule:
**macro readability beats micro cleverness**.

---

## The show must stop behaving like one object in front of a background

The frame must be treated like a staged image with multiple authorities.

### Image authorities

At different moments, the main authority should shift between:

- the hero
- the chamber
- the environmental field
- the accent / tactical event layer
- the post / residue layer

The strongest version of VisuLive will not ask the hero to do every job.

The hero can remain iconic.
But it cannot remain the only meaningful actor.

---

## The visual grammar we should lean into harder

### Dark-anchor plus emitted structure

The most promising direction so far is clear:

- dark mass
- electric seams
- rim light
- fresnel energy
- emitted lattice / membrane / edge behavior

The show gets weaker whenever it falls back into:

- a softly lit filled object
- warm/cool sculpture lighting as the main image grammar
- too much polite glow with not enough decisive structure

### Screen-space consequence

The show needs more true image-space consequence.
Not just brighter materials.

That means stronger use of:

- selective bloom
- additive flare / halo spend
- afterimage and residue
- chromatic separation at impact moments
- directional stains
- shockwave-like screen-space travel
- occlusion-aware light ghosts
- event-limited distortion

### Authored phrase logic

We need more authored cueing above detection.

Not every strong moment is a “drop.”
Different moments deserve different classes of spend:

- inhale / pre-ignition
- snap / tactical accent
- structural widening
- structural collapse
- shell peel / recoil
- halo ignition
- laser fan deployment
- world-field activation
- residue / haunt / stain

### Big-screen composition

Everything must be judged as if it will live on a large display.
That means:

- more negative space discipline
- more compositional widen / tighten events
- less constant center-locking
- less medium-intensity sameness
- stronger silhouette change
- more respect for the screen edges
- more peripheral participation

---

## Anti-goals

The following should be treated as real failure modes:

- parameter soup
- one-scene louder/slower/faster instead of authored state changes
- constant medium-brightness mush
- random color spending without hierarchy
- equalizer tropes and EDM cliches
- constant “doing things” without tension or release
- detail that only rewards a close-up view
- cue detection that fires more often than the show can spend convincingly
- coding progress that does not materially improve the image

---

## What Codex is best used for

Codex should not be asked to “figure out the art” in a vacuum.

Use Codex primarily for:

- architecture splits
- subsystem extraction
- clean contracts between conductor, cue engine, choreography, render, and operator layers
- implementing a designed cue taxonomy
- integrating post passes
- building capture / analysis / tooling loops
- creating experimental branches from explicit briefs
- writing reusable scene systems once the image language is already chosen

Codex should **not** be treated as the final source of:

- taste
- visual dramaturgy
- aesthetic selection pressure
- what references matter and why
- what “good enough for a big screen” means

That layer should come from a stronger creative brief and a harder integrator.

## What references are worth adding and why

Do not add references because they are famous.
Add them because they teach a specific missing principle.

### A. Browser / shader-native intensity

Study when the goal is:

- high visual spend on the web
- multi-pass thinking
- feedback / residue / image-space logic
- bold contrast and color authority

References:

- Butterchurn
- Synesthesia
- strong Shadertoy / shader-art pieces with iconic silhouettes and limited vocabularies

What to steal:

- confidence
- signal-to-style efficiency
- multi-pass boldness
- how much can be achieved with screen-space logic, not just 3D geometry

### B. Live-show systems

Study when the goal is:

- real musical cueing
- operator reality
- layer compositing
- show pacing
- production-grade consequence

References:

- Notch
- Resolume + Wire
- TouchDesigner
- vvvv gamma

What to steal:

- modular signal graphs
- cue hierarchy
- exposed-parameter discipline
- image compositing vocabulary
- performance-minded iteration

### C. Game VFX / boss-fight staging

Study when the goal is:

- iconic silhouette language
- impact readability
- screen-space hit language
- emotional consequence
- layering of anticipation, impact, and aftermath

References:

- Control
- Returnal
- Destiny raid/boss VFX language
- Diablo IV high-end VFX readability
- FFXIV raid telegraph / arena-event logic

What to steal:

- telegraphed authority
- arena-scale event language
- iconic pre-impact states
- residue and corruption after big spends

### D. Music-driven authored experiences

Study when the goal is:

- phrase-scale transformation
- sensory escalation
- music-native pacing
- how repetition becomes ritual instead of boredom

References:

- Rez Infinite
- Tetris Effect
- Thumper
- Child of Eden

What to steal:

- gameplay-grade cue clarity
- emotionally legible escalation
- clean iconic motifs that transform with the music

### E. Concert lighting / laser language

Study when the goal is:

- room command
- dramatic cue reveals
- energy discipline
- black-value respect
- huge perceived scale from simple forms

References:

- grandMA show programming mindset
- laser fan / tunnel / sheet composition
- moving-head fixture choreography
- reveal-and-withhold pacing from live shows

What to steal:

- darkness as a design tool
- cue stacks
- executors / scenes / looks mentality
- simple shapes with overwhelming authority

## Missing image vocabularies that should become first-class systems

These are strong candidates for explicit implementation lanes.

### 1. Event composites

A layer that exists to spend moments, not to idle.

Examples:

- impact rings
- halo ignition
- additive lens streaks
- directional color tears
- field contamination
- bloom bursts
- transient edge ghosts

### 2. Residue system

A layer that remembers what just happened.

Examples:

- afterimages
- stains
- shell scars
- light ghosts
- delayed field decay
- corrupted palette remnants

### 3. Chamber authority system

A layer that lets the world become the event.

Examples:

- vault lines
- cathedral rings
- portal sheets
- volumetric fans
- peripheral light architecture
- deep-space movement that changes scale perception

### 4. Cue taxonomy layer

A layer above detection that decides *what class of event this is*.

Examples:

- accent
- gate-open
- build-gather
- false peak
- real peak
- drop-hit
- phrase turn
- release tail
- aftermath bloom
- quiet haunt

### 5. Framing director

A layer that treats camera and composition as authored show behavior, not just parameters.

Examples:

- widen on pressure
- compress before impact
- hold after hit
- drift out during aftermath
- change hero dominance by intent, not by random noise

## Acceptance criteria for the next real leap

The next round of work should be judged against these, not “it looks cooler to us.”

### A. Big-screen authority

Can a still frame or five-second clip read as intentional, premium, and commanding from across a room?

### B. Cue consequence

Do strong musical moments produce distinct classes of consequence, not just more intensity?

### C. Image variety without soup

Can the show transform meaningfully while still feeling like one authored system?

### D. Dark-value discipline

Do blacks remain valuable enough that event spending feels expensive?

### E. Whole-frame participation

Does the full screen help tell the story, not just the center?

### F. Aftermath

Do strong moments leave residue, posture shifts, or memory?

### G. Consistency

Do similar moments produce similarly satisfying categories of result?

If the answer is “mostly no,” the pass is not yet the leap.

## Immediate recommendations for the next Codex manager cycle

### 1. Do not start with more look tuning

Start with structure.

- split the scene monolith
- isolate hero / chamber / event composite / direction / choreography responsibilities
- create a cue taxonomy contract above raw conductor signals

### 2. Add a real event-composite lane

This should be a dedicated layer, not an afterthought.

### 3. Add a residue / memory lane

Big shows do not just hit.
They linger.

### 4. Make framing an authored system

Not just one more slider.

### 5. Retest using curated reference goals

Every pass should be able to answer:

- which reference principle was targeted?
- what changed in the image?
- how would a viewer feel the difference?

### 6. Judge the show in clips, not just live tweaking

The project now needs more “watch the result back” discipline.

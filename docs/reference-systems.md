# Reference Systems

This project should keep borrowing principles from mature show tools and visualization systems without copying their visual identity.

Use this file as a principle router.
Use [flagship-reference-atlas.md](C:/dev/GitHub/visulive/docs/flagship-reference-atlas.md) for the labeled board and deeper extraction notes.

## Cue and playback grammar

- [`grandMA3`](https://help.malighting.com/grandMA3/2.3/HTML/cue_sequence.html)
- [`grandMA3 cue timing`](https://help.malighting.com/grandMA3/2.3/HTML/cue_timing.html)
- [`grandMA3 phasers`](https://help.malighting.com/grandMA3/2.3/HTML/phaser.html)
- [`Resolume Wire FFT`](https://resolume.com/support/en/wire-fft)

What to steal:

- cue hierarchy
- timing hierarchy
- phase thinking
- sequence/playback discipline
- repeatable archetypes instead of parameter soup

What not to steal:

- literal console UX
- preset soup
- one global transition value for everything

## Screen-space consequence

- [`three.js WebGPURenderer`](https://threejs.org/docs/#manual/en/introduction/WebGPURenderer)
- [`three.js TSL`](https://threejs.org/docs/TSL.html)
- [`AfterImageNode`](https://threejs.org/docs/pages/AfterImageNode.html)
- [`Disguise compositing layers`](https://help.disguise.one/designer/layers/compositing-layers)

What to steal:

- afterimage and residue as first-class tools
- bloom and flare as selective spends
- layer/mask thinking
- WebGPU-native post composition

What not to steal:

- always-on smear
- uniform bloom fog
- a flattened single render path

## Audio reduction and timing

- [`Notch working with audio`](https://manual.notch.one/1.0/en/docs/learning/working-with-audio/)
- [`TouchDesigner Audio Spectrum CHOP`](https://docs.derivative.ca/Audio_Spectrum_CHOP)
- [`TouchDesigner Timer CHOP`](https://docs.derivative.ca/Timer_CHOP)

What to steal:

- broad envelopes plus precise triggers
- timed-process thinking
- reusable signal reduction
- separation between analysis and consequence

What not to steal:

- raw FFT dumped directly into visuals
- stateless immediate reactions

## Browser-native intensity

- [`Synesthesia docs`](https://synesthesia.live/docs)
- [`Synesthesia Shader Format`](https://app.synesthesia.live/docs/ssf/ssf.html)
- [`projectM`](https://github.com/projectM-visualizer/projectm)
- [`Butterchurn`](https://github.com/jberg/butterchurn)

What to steal:

- shader-first boldness
- strong preset archetypes
- confident transitions
- browser-native intensity without apology

What not to steal:

- rainbow soup
- dull web-demo restraint

## VFX architecture

- [`Unreal Niagara overview`](https://dev.epicgames.com/documentation/fr-fr/unreal-engine/overview-of-niagara-effects-for-unreal-engine)
- [`vvvv gamma`](https://vvvv.org/download/)
- [`vvvv gamma 3D graphics / VL.Stride`](https://thegraybook.vvvv.org/reference/libraries/graphics-3d.html)
- [`vvvv gamma audio`](https://thegraybook.vvvv.org/reference/libraries/audio.html)
- [`vvvv gamma RenderDoc GPU debugging`](https://vvvv.org/blog/2024/introducing-renderdoc-gpu-debugging/)

What to steal:

- system/emitter/module thinking
- control-data versus image-data separation
- profiling discipline
- modular show logic

What not to steal:

- one giant scene file forever
- guess-first tuning

## World-building and staging

- `teamLab Borderless`
- `Moment Factory`
- `Refik Anadol`
- `Rez Infinite`
- `Tetris Effect`
- `Thumper`
- `Control`
- `Returnal`

What to steal:

- world participation
- room-scale consequence
- silhouette and timing discipline
- residue after impact
- stage authority from composition, not just effect count

What not to steal:

- formless AI slush
- wallpaper spectacle
- one-shot surprise without aftermath

## What this means for VisuLive now

The next pass should spend on:

- cue hierarchy
- screen-space consequence
- residue and memory
- modular effect families
- chamber participation
- dark-value discipline
- archetype contrast

The next pass should avoid:

- preset soup
- random rainbow spam
- sculpture-lighting habits
- overbright baseline haze
- tiny reactive noise everywhere
- a hero that monopolizes every frame forever

For the labeled board and per-reference extraction notes, use [flagship-reference-atlas.md](C:/dev/GitHub/visulive/docs/flagship-reference-atlas.md).

# VisuLive Anthology Mastery Charter

Date: 2026-04-22
Status: Active mastery charter

This document defines the uncompromising end-state for VisuLive.

It exists so future agents do not have to infer the dream from thread history,
taste alone, or a pile of partial plans.

## The Core Identity

VisuLive is:

- one simple public portal
- one autonomous `Start Show` path
- one optional `Advanced` layer
- one deep internal anthology engine

The surface stays simple on purpose.
The internal capability should keep getting deeper, stranger, more memorable,
and more authored over time.

## What Mastery Means Here

Mastery for VisuLive means:

- the untouched show is already worth watching
- the music changes the ontology of the frame, not just its intensity
- hero, chamber, world, light, particles, consequence, and memory can each own the picture
- silence, low-energy passages, and recovery states remain alive
- the browser-native renderer still feels premium on `webgpu / safe`
- the system contains more than one unforgettable image grammar
- recurrence feels authored instead of repetitive
- proof can explain why a family deserves to stay, graduate, or be retired

The dream is not a prettier settings app.
The dream is a deeper autonomous show intelligence.

## What Is Sacred

These rules are sacred:

- `Start Show` stays the front door
- route choice remains the only required pre-show choice
- untouched no-touch quality stays the primary product test
- public simplicity is a product rule, not a creative limit
- internal anthology expansion is the ambition rule
- safe tier is a real authored target, not an apology path
- consequence must feel earned
- memory must eventually become visible and deliberate
- proof outranks preference when deciding if a family is real

## What Is Disposable

These things are disposable if they limit the dream:

- legacy quick-start ideology
- slider-first thinking
- one-scene-forever language
- any current family that cannot justify its cue role
- any architecture that keeps policy and render ownership tangled
- any effect that only adds noise, heat, or clutter
- any menu surface that asks the user to do the show’s job

## What Counts As Compromise

The following are compromise signs, not progress:

- the show needs operator intervention before it becomes compelling
- more named options appear, but the no-touch run is not better
- a new family exists only in metadata or UI without image-class value
- impact still means brightness instead of changed authorship
- quiet states die or become empty while peaks get louder
- safe tier reads like a degraded fallback
- the same hero or same chamber answer keeps winning every cue
- mixed media lands as collage instead of authored language
- repetition is mistaken for memory

## The Long-Term End-State

The long-term target is:

- a browser-native anthology engine
- with multiple hero species
- multiple world grammars
- mutation verbs
- consequence and aftermath families
- lighting rigs and camera phrases
- particle field roles
- mixed-media compositing
- motif and memory behavior
- deeper music semantics

The public product can still remain one portal while that engine grows.

## The Product Law

The public product should always read as:

- choose `PC Audio`, `Microphone`, or `Combo`
- press `Start Show`
- trust the system first
- open `Advanced` only when you want curation, steering, repair, capture, replay, or diagnostics

The public product should never require:

- scene browsing before the show starts
- a preset soup
- broad front-door control walls
- hidden narrowing defaults that weaken the autonomous run

## The Capability Law

Internal growth should happen through capability families, not public clutter.

The permanent capability families are:

- Hero Ecology
- World Grammar / Mutation
- Consequence / Aftermath / Post
- Lighting / Cinematography
- Particles / Fields
- Mixed Media / Compositor / Content
- Motif / Memory
- Music Semantics

Every family must eventually prove:

- quiet value
- impact value
- aftermath value
- safe-tier dignity
- no-touch usefulness
- clear absence rules

## The Review Law

The show is not allowed to drift by novelty alone.

Every serious wave should leave behind:

- an updated capability map
- an updated extraction scoreboard if ownership changed
- proof expectations tied to the graduation rubric
- a checkpoint summary for the next agent

## Bottom Line

VisuLive should become a show people want to watch even when nobody is touching it.

If a pass makes the menu richer but the no-touch show flatter, the pass is not
good enough.

If a pass deepens repertoire, strengthens authority shifts, improves silence,
and makes the show more unforgettable without adding public clutter, it is
moving in the right direction.

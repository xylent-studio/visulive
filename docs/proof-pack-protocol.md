# VisuLive Proof Pack Protocol

Status: Active

This document defines how VisuLive validation packs are built and judged. It is a fresh-batch-only protocol.

## Purpose

Proof packs exist to answer one question: did the latest system changes improve the show in a way that is visible, distinct, and worth keeping?

A proof pack must prioritize:
- whole-frame authority
- cue separation
- chamber/world participation
- safe-tier readability
- before/during/after consequence
- residue or memory when the cue calls for it
- motion phrases that are visibly more than one-axis wobble
- palette holds and handoffs that read as act/family choices

## Scope

This protocol applies to:
- new inbox capture batches collected after the current reset point
- curated stills chosen from those batches
- analyzer summaries derived from those batches
- acceptance notes written against those batches

This protocol does not authorize tuning decisions from stale inbox batches.

## Fresh Batch Rule

1. A fresh batch is the current capture set collected after the last deliberate evidence reset.
2. Only the active fresh batch may drive tuning, selection, and acceptance calls.
3. Historical batches may be referenced for comparison, but they must not be used as the primary basis for current direction.
4. If a batch predates the current work cycle, it is reference-only unless explicitly reactivated.

## What Counts As Useful Evidence

Useful evidence must help answer at least one of these questions:
- did the cue create a distinct image class
- did the whole frame change, not just the hero
- did the chamber/world participate meaningfully
- did the frame read at distance
- did the system preserve silhouette and composition under load
- did the cue produce a clear before/during/after sequence
- did safe-tier remain premium and legible
- did residue, afterimage, or subtraction appear when the cue demanded it
- did the motion read as a spatial phrase instead of a single-axis spin
- did the palette change feel authored by act or family

Evidence is strongest when it shows:
- a clear cue family label
- a clear dominant actor
- a clear change in frame scale or spatial authority
- a clear contrast between peak and aftermath
- a clear success and failure pair from the same pass

## Strong Still Selection Rules

Strong stills must be chosen when they satisfy most of the following:
- the cue family is readable without guessing
- the chamber or frame architecture is doing real work
- the hero is either intentionally dominant or intentionally restrained
- the silhouette is intact enough to read from a room
- the composition has a strong foreground, midground, or framing decision
- the frame shows a meaningful shift from nearby frames
- the image does not collapse into washout, mush, or generic glow
- any residue present feels earned, not accidental

Prefer strong stills that show:
- a peak moment with preserved structure
- a transition moment that clearly changes the scene state
- an aftermath frame with visible residue or altered space
- a chamber-dominant frame where the hero is not doing all the work
- a motion read that shows bank, roll, arc, or precession as intended
- a palette read that makes the act or family choice obvious

## Weak Still Selection Rules

Weak stills must be kept when they are useful for diagnosis. Select them when they show:
- the right cue family but the wrong visual authority
- hero-only spend with weak chamber participation
- overbright or overwashed composition
- unclear silhouette at distance
- repeated frames that do not add new information
- residue that reads as haze instead of memory
- a frame that proves the system needs restraint, subtraction, or better routing

Weak stills are valuable when they explain what failed. They are not valuable when they merely repeat the same failure.

## Pack Composition

Each proof pack should include:
- 3 to 5 strong stills
- 3 to 5 weak stills
- at least one before/during/after chain when possible
- cue labels for every still
- a short note for why each still was selected
- a short note for what changed relative to the previous pass
- any analyzer metrics that materially explain the image

## Review Criteria

A proof pack passes review only if it can show:
- cue classes create distinct frame classes
- chamber/world authority rises when the cue demands it
- hero scale changes are intentional, not random
- screen-space consequence is selective, not constant
- safe-tier remains premium enough to ship
- aftermath is visibly different from peak

## What Is Not Useful Evidence

Do not use stills that only show:
- brightness without structure
- motion blur without readable state
- duplicate frames with no new information
- late-cycle overprocessed wash
- stale historical captures as if they were current truth
- generic “nice looking” frames that do not answer a question

## Required Review Notes

Every proof pack review must state:
1. what improved
2. what regressed
3. what is still overcommitted
4. what cue families now look distinct
5. what should be retested in the next fresh batch
6. whether motion, palette, and screen-space consequence actually changed the image

If the answer depends on an old batch, the note must say so explicitly and mark that batch as reference-only.

# VisuLive Proof Review Notes

Use this folder for short review notes tied to meaningful fresh proof waves.

Recommended flow:

1. collect the fresh batch into [captures/inbox](C:/dev/GitHub/visulive/captures/inbox)
2. tag each serious run with a proof scenario in diagnostics
3. run `npm run proof:current`
4. scaffold a note with `npm run proof:review-note`
5. finish the human verdict for `truth`, `hierarchy`, `coverage`, `taste`, and `operator trust`
6. only then decide whether any capture should be promoted to `current-candidate` or `current-canonical`

Do not treat this folder as a screenshot dump.
It is the durable human judgment layer that sits on top of the analyzer and proof pack.
